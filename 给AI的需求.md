# Role

你是一位精通 Python 开发的专家，熟悉 `uv` 工具链和跨平台 CLI 工具开发。

# Project Goal

请创建一个基于 Python `uv` 的命令行工具项目。

- **项目名称**: `qrcode49`
- **核心功能**: 根据输入文件类型，自动在“生成二维码”和“解析二维码”之间切换。

# Tech Stack & Strict Requirements

1. **项目管理**: 全程使用 Python `uv` 进行初始化和依赖管理。
   可以作为 uv tool 包安装
2. **严格约束**:
   - **禁止**使用系统包管理器（如 `apt`, `yum`, `brew` 等）安装任何依赖。
   - 所有依赖必须通过 `uv add` 安装，且必须选择提供二进制包的纯 Python 库，确保在 Windows/Linux/macOS 上无需编译即可直接运行。

# Core Logic (CLI Interface)

**命令名称**: `qrcode49`
**执行格式**: `qrcode49 <文件路径1> <文件路径2> 边长`

程序需根据 `<文件路径1>` 的后缀自动判断任务：

**情况 A：生成二维码 (文本 -> 图片)**

- **条件**: `<文件路径1>` 是文本文件 (`.txt`, `.md`)。
- **动作**: 读取文本内容，生成二维码图片并保存到 `<文件路径2>`。
- **支持格式**: 输出支持 `.png`, `.jpg`, `.webp`。

**情况 B：解析二维码 (图片 -> 文本)**

- **条件**: `<文件路径1>` 是图片文件 (`.png`, `.jpg`, `.webp`)。
- **动作**: 解析图片中的二维码内容，将文本写入到 `<文件路径2>` 中。

**关于边长**

生成二维码时，在两个文件路径最后可以添加参数边长，不添加也行，边长是纯数字，边长越大二维码越大，反之一样
